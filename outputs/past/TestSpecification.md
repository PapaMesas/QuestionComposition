## 3. 単体テスト (Unit Tests)

### 3-1. `config/crypto.rs` — 暗号化・復号

| ID | テスト内容 | 合格条件 |
|---|---|---|
| UT-01 | `encrypt("test_api_key")` が非空文字列を返す | `Ok(s)` で `s` が空でない |
| UT-02 | `decrypt(encrypt("test_api_key"))` が元の値を返す | 復号値 == `"test_api_key"` |
| UT-03 | 不正な暗号文を `decrypt` に渡した場合にエラーを返す | `Err(_)` を返す |
| UT-04 | 異なる入力値を暗号化した場合に異なる暗号文が生成される | `encrypt("a") != encrypt("b")` (確率的) |

### 3-2. `excel/reader.rs` — Excel 読み込み

| ID | テスト内容 | 合格条件 |
|---|---|---|
| UT-05 | 正常な .xlsx を読み込んで科目名を取得できる | `sheet.subject` が期待値と一致 |
| UT-06 | 正常な .xlsx を読み込んで設問数が正しい | `sheet.questions.len()` が期待値と一致 |
| UT-07 | 各設問の `question_no`・`body`・`correct_answer` が正しく取り込まれる | 各フィールドが期待値と一致 |
| UT-08 | デフォルトの `num_choices` が 4 である | `q.num_choices == 4` |
| UT-09 | 空行が自動スキップされる | 空行を含む xlsx でも設問数が正しい |
| UT-10 | 存在しないパスを渡した場合にエラーを返す | `Err(_)` を返す |

### 3-3. `generator.rs` — JSON 配列パース

| ID | テスト内容 | 合格条件 |
|---|---|---|
| UT-11 | 正常な JSON 配列文字列を解析できる | `["a","b","c"]` → `vec!["a","b","c"]` |
| UT-12 | コードブロック内の JSON 配列を解析できる | `` ```json\n["a"]\n``` `` → `vec!["a"]` |
| UT-13 | JSON 配列がない文字列を渡した場合にエラーを返す | `Err(_)` を返す |
| UT-14 | 模範解答がランダム位置に挿入される | N回試行して挿入位置が複数パターンになる |
| UT-15 | 挿入後の選択肢数が `num_choices` と一致する | `choices.len() == num_choices` |

### 3-4. `rule_loader.rs` — ルール読み込み

| ID | テスト内容 | 合格条件 |
|---|---|---|
| UT-16 | デフォルトルールが空でない | `rule_set.content` が非空 |
| UT-17 | カスタム .md ファイルを読み込める | `content` がファイル内容と一致 |
| UT-18 | 存在しないファイルを渡した場合にエラーを返す | `Err(_)` を返す |
