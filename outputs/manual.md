# 操作マニュアル — 設問作成支援ツール

---

## 概要

本ツールは、Excel形式の設問ファイルを読み込み、LLM（Gemini・ChatGPT・Claude）を使って多岐選択肢問題の誤答選択肢を自動生成するデスクトップアプリケーションです。

日本語フォントとして **Noto Sans JP Medium**（OFLライセンス）をバイナリに埋め込んでいるため、日本語が正しく表示されます。

---

## 起動方法

```bash
# プロジェクトディレクトリで実行する
cd questionComposition
cargo run --release
```

ウィンドウが起動したら、上部のタブを順番に操作します。

---

## 操作手順

### ① API設定タブ

LLMのAPIキーを登録します。**最初に必ず設定してください。**

1. 「LLM プロバイダー」ドロップダウンから使用するサービスを選ぶ
   - `Gemini`（Google）
   - `ChatGPT (OpenAI)`
   - `Claude (Anthropic)`
2. 「API キー」欄にAPIキーを入力する
   - 「表示」ボタンで入力内容を確認できる
3. 「APIキーを保存」ボタンを押す
4. 「✓ APIキーが登録されています」（緑）が表示されれば登録完了

> **注意**: APIキーは暗号化されてローカルファイルに保存されます。アプリを再起動しても再入力は不要です。

---

### ② 設問取り込みタブ

設問データが入ったExcelファイルを読み込みます。

1. 「📂 Excelファイルを選択 (.xlsx)」ボタンを押す
2. ファイル選択ダイアログで対象の `.xlsx` ファイルを選ぶ
3. 「✓ 取り込み完了: 〇〇 N問」が表示されれば成功

#### 入力ファイルのフォーマット

| 行 | 内容 |
|---|---|
| 1行目 | 科目名（A列） |
| 2行目 | ヘッダー（No, 問題番号, 問題本文, 選択肢数, 模範解答） |
| 3行目以降 | データ行 |

| 列 | 内容 |
|---|---|
| A列 | 通し番号（1から始まる整数） |
| B列 | 問題番号（例: 1-1） |
| C列 | 問題本文 |
| D列 | 選択肢数（空欄でも可） |
| E列 | 模範解答 |

---

### ③ 選択肢数設定タブ

各設問の選択肢数を設定します。

1. 取り込んだ全設問が一覧表示される
   - 問題本文は250文字以内は全文、超える場合は先頭250文字を表示
2. 各行の「選択肢数」ドロップダウンで選択肢の数を変更する（2〜8、デフォルト: 4）
3. 全設問を確認したら、末尾の「設定が完了しました（全設問の選択肢数を確認済み）」チェックボックスをオンにする

---

### ④ ルール設定タブ

選択肢生成に使うガイドラインを設定します。

- **デフォルト**: `InputMaterials/` 内の2つのルールファイルが自動で読み込まれます
- **カスタムルール**: 「📄 カスタムルール (.md) を読み込む」ボタンで任意の `.md` ファイルを指定できます
- **リセット**: 「デフォルトルールに戻す」ボタンで元に戻せます

> ルール本文はプレビュー欄でスクロールして確認できます。

---

### ⑤ 作問生成タブ

選択肢の自動生成と結果の保存を行います。

#### 「▶ 作問開始」ボタンが有効になる条件

以下の3つすべてが満たされた場合にのみボタンが押せます：

- ✅ APIキーが登録済み
- ✅ 設定完了チェックボックスがオン
- ✅ 設問ファイルが取り込み済み

#### 生成手順

1. 「▶ 作問開始」ボタンを押す
2. 生成中はスピナーと進捗（`N/M`）が表示される
3. 完了後、「✓ 生成完了: N問」が表示される
4. 「💾 結果を Excel (.xlsx) に保存」ボタンを押す
5. 保存先を選択して保存する

#### 出力ファイルのフォーマット（Excel）

| 行 | 内容 |
|---|---|
| 1行目 | 科目名 |
| 2行目 | ヘッダー（No, 問題番号, 問題本文, 模範解答, 選択肢数, 選択肢1, 選択肢2, …, **ガイドライン**） |
| 3行目以降 | データ行 |

> 模範解答は選択肢のうちランダムな位置に配置されます。
> **ガイドライン列**: 選択肢生成に使用した `test_guideline.md` のガイドライン番号がカンマ区切りで記載されます（例: `21, 27, 33`）。

---

### ⑥ エクスポートタブ

生成した設問を複数の形式でエクスポートします。

#### エクスポート形式の選択

以下の5形式から出力形式を選択できます：

1. **Moodle XML** - Moodle のネイティブ形式
2. **QTI 1.2** - 古いバージョンの IMS QTI 標準
3. **QTI 2.1** - IMS QTI 標準（最初の2.x版）
4. **QTI 2.2** - IMS QTI 標準（2.1 のマイナーアップデート）
5. **QTI 3.0** - IMS QTI 標準（最新版）

#### エクスポート手順

1. 「⑥ エクスポート」タブを開く
   - 生成済みの設問がある場合のみ利用可能です
2. 出力形式をラジオボタンから選択する
   - ファイル名が自動で形式に合わせて更新されます（例: `output.xml`）
3. 「📂 出力先を選択」ボタンを押す
4. 保存先フォルダを選択する
5. 「📤 エクスポート」ボタンを押す
6. 「✅ エクスポート完了」が表示されたら成功

#### ファイルについて

- **Moodle XML**: Moodle のクイズ機能に直接インポート可能
  - CDATA 形式のテキスト、defaultgrade・penalty・shuffleanswers 等の Moodle 標準フィールドを含む
  - 正答は `fraction="100"`、誤答は `fraction="0"` で出力される
- **QTI ファイル**: 対応する学習管理システム（LMS）にインポート可能
  - QTI 1.2: 古いシステム向け（`render_choice`・アルファベット ident・スコア 0-1）
  - QTI 2.1: モダンなシステム向け（`responseDeclaration`・`match_correct` テンプレート）
  - QTI 2.2: QTI 2.1 の更新版（purl.imsglobal.org スキーマ定義）
  - QTI 3.0: 最新システム向け（`qti-` プレフィックス要素・アルファベット ident・ハイフン属性）

> **ガイドライン情報**: Moodle XML では `<generalfeedback>` に、QTI 各形式では対応するメタデータフィールドに、使用したガイドライン番号が含まれます。

## よくあるエラーと対処

| エラーメッセージ | 原因 | 対処 |
|---|---|---|
| `✗ APIキーが未登録です` | APIキーが保存されていない | ①タブでAPIキーを保存する |
| `✗ 読み込みエラー` | .xlsxのフォーマットが不正 | ファイルのフォーマットを確認する |
| `✗ APIキーの読み込みに失敗しました` | 設定ファイルが壊れている | ①タブでAPIキーを再登録する |
| `⚠ N問でエラー` | 一部の設問でLLM呼び出しに失敗 | APIキーの有効性・ネットワーク接続を確認する |
